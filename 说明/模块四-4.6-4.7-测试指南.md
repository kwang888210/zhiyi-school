### 1.3 SQL 验证

```sql
-- 会话列表接口（后端视角）
-- 应该能查到 admin 参与的所有 conversation
SELECT DISTINCT conversation_id, 
       COUNT(*) AS total_msgs,
       SUM(CASE WHEN receiver_id = 1 AND is_read = 0 THEN 1 ELSE 0 END) AS unread_to_admin
FROM chat_message 
WHERE sender_id = 1 OR receiver_id = 1
GROUP BY conversation_id;
```

## 二、4.7 内容强制管理

### 2.1 强制下架商品

**准备数据**：

```sql
-- 创建一个测试商品（假设发布者 ID=2，分类 ID=1 数码电子）
INSERT INTO item (publisher_id, type, title, description, category_id, price, images, status, ai_reviewed)
VALUES (2, 'SELL', '测试商品-待强制下架', '这是一个测试商品', 1, 99.00, '[]', 'ON_SALE', 1);

-- 记下商品 ID
SELECT id, title, status FROM item WHERE title LIKE '%待强制下架%';
```

**测试步骤**：

| 步骤 | 操作 | 期望 |
|------|------|------|
| 1 | admin → 🔧 内容管理 | 页面显示两个工具卡片 |
| 2 | 在"强制下架商品"输入商品 ID → 查询商品 | 预览卡片显示商品标题、状态、价格 |
| 3 | 点击"确认下架" → 弹窗二次确认 → 确认 | 提示"商品已强制下架" |
| 4 | 再次查询同一商品 ID | 状态变为"已下架"（OFF_SHELF） |

**SQL 验证**：

```sql
-- 1. 商品状态应为 OFF_SHELF
SELECT id, title, status FROM item WHERE title LIKE '%待强制下架%';

-- 2. 卖家经验应 -30
SELECT id, nickname, exp FROM sys_user WHERE id = 2;

-- 3. 处罚日志应有一条 WARNING 记录
SELECT vl.*, u.nickname AS punished_user 
FROM violation_log vl 
JOIN sys_user u ON vl.user_id = u.id 
ORDER BY vl.created_at DESC LIMIT 1;
-- type 应为 WARNING，reason 应包含"强制下架"
```

### 2.2 强制重置密码

**测试步骤**：

| 步骤 | 操作 | 期望 |
|------|------|------|
| 1 | 在"强制重置密码"区域输入学号或昵称 → 搜索 | 搜索结果列表出现对应用户 |
| 2 | 点击选中用户 | 用户卡片高亮，显示预览信息 |
| 3 | 点击"确认重置密码" → 弹窗确认 | 提示"密码已重置为 123456" |
| 4 | 用该用户账号 + 密码 `123456` 登录 | 登录成功 |
| 5 | 用该用户旧密码登录 | 登录失败 |

**SQL 验证**：

```sql
-- Token 版本应递增
SELECT id, nickname, token_version FROM sys_user WHERE id = <被重置用户ID>;
-- token_version 应比之前 +1
```

### 2.3 边界情况测试

| 场景 | 操作 | 期望 |
|------|------|------|
| 重复下架 | 对已 OFF_SHELF 商品再次下架 | 提示"该商品已处于下架状态" |
| 重置管理员密码 | 搜索 admin → 重置密码 | 提示"不能重置管理员密码"或按钮禁用 |

| 不存在的商品 | 输入不存在的商品 ID → 查询 | 提示"商品不存在" |
